# 创业者48小时生存实验

黑镜风格的互动创业叙事游戏。你刚发出辞职信，账户只剩一点钱，房租倒计时已经开始。你能在 48 小时内活下来吗？

## 项目简介
这是一款纯前端互动游戏，用资源管理、分支选择、道德代价和结局传播，去模拟“决定创业之后的前 48 小时”。

它的重点不是传统游戏系统深度，而是：
- 强叙事体验
- 强情绪压迫
- 移动端优先
- 强分享传播
- 低部署成本

## 当前版本亮点
- 首页、打字页、主界面、结尾页、分享预览页已经完整跑通
- 4 维资源系统：资金、时间、精力、人脉
- 资源会真实影响选项可用性
- 资源危机时会有界面紧张反馈
- 最后进入结尾页前有“收束坍缩”过渡
- 已接入 BGM、打字音效、按钮/选择音效
- 分享预览图已改成更偏“结果海报”的表达

## 玩法概览
1. 首页进入产品
2. 打字页铺垫情绪
3. 点击“启动实验”后进入主游戏
4. 在不同节点做选择，消耗或获得资源
5. 先看 consequence 结果反馈，再继续推进
6. 进入结尾页，生成分享图并传播

## 当前体验规则

### 资源
- 资源维度：资金、时间、精力、人脉
- 时间耗尽会结束游戏
- 金钱 / 精力 / 人脉归零不一定立刻结束，但可能把你推入“无路可走”
- 某些选项会因为资源不足直接不可选

### 危机反馈
- 资源进入 `critical` 时会有更强的界面紧张感
- 核心资源第一次归零时会触发更明显的震动与故障反馈
- 这些反馈只增强感知，不改变主流程

### 音频
- 打字页会播放打字音效
- BGM 不会在首页自动开始
- 只有点击“启动实验”后才开始播放 BGM
- 进入结算页时会停止 BGM

### 分享
- 分享图不再使用资源条，而是使用首页人物主视觉
- 分享链接复制地址为：
  - `https://startup-48h-survival-game.vercel.app/`
- 分享二维码来自：
  - `assets/qrcode.png`

## 本地运行
直接起一个静态服务即可：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 部署
当前线上正式地址：

```text
https://startup-48h-survival-game.vercel.app/
```

如果使用 Vercel：
- 需要把代码先 `commit`
- 再 `push origin main`
- Vercel 才会基于 GitHub 新提交自动部署

注意：
- 只 `git add` 不 `git commit`，Vercel 不会更新
- 静态资源改动后如果看到旧版本，先检查缓存版本号和部署状态

## 当前维护原则

- 优先保护现有体验
