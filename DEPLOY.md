# 快速部署指南

## 方式1：Vercel部署（推荐，最快）

### 步骤：
1. 访问 https://vercel.com
2. 使用GitHub账号登录
3. 点击 "New Project"
4. 导入这个项目文件夹
5. 点击 "Deploy"
6. 等待30秒，获得线上地址

**优点**：
- 完全免费
- 自动HTTPS
- 全球CDN加速
- 部署时间：< 1分钟

---

## 方式2：Netlify部署（拖拽上传）

### 步骤：
1. 访问 https://app.netlify.com/drop
2. 直接把整个项目文件夹拖进去
3. 等待上传完成
4. 获得线上地址

**优点**：
- 无需注册（可选）
- 拖拽即可
- 部署时间：< 30秒

---

## 方式3：GitHub Pages（免费托管）

### 步骤：
1. 在GitHub创建新仓库
2. 上传所有文件
3. 进入仓库设置 → Pages
4. 选择 main 分支
5. 保存，等待部署

**优点**：
- 完全免费
- 稳定可靠
- 部署时间：2-3分钟

---

## 部署后必做的3件事

### 1. 修改小助手微信号

编辑 `js/game.js` 第 200 行左右：

```javascript
function contactAssistant() {
  alert('添加小助手微信：[改成你的微信号]\n\n我们会给你更详细的创业建议和反馈。');
}
```

### 2. 更新公众号文章中的游戏链接

编辑 `article.md`，把 `[游戏链接：请在此处插入实际部署后的链接]` 改成实际地址。

### 3. 测试所有路径

在浏览器中打开游戏，测试：
- 开场动画是否正常
- 6个节点是否都能正常跳转
- 资源条是否正常显示和更新
- 6种结局是否都能触发
- 分享功能是否正常

## 常见问题

### Q: 游戏打开是空白页？
A: 检查浏览器控制台是否有JS错误，确保所有文件都已上传。

### Q: 资源条不显示？
A: 清除浏览器缓存，刷新页面。

### Q: 分享功能不工作？
A: 部分浏览器不支持Web Share API，会自动降级为复制文案。

### Q: 移动端显示异常？
A: 检查是否有CSS文件缺失，确保响应式样式生效。

## 数据监控

### 通过浏览器控制台查看数据：

```javascript
// 查看所有游戏记录
JSON.parse(localStorage.getItem('gameResults'))

// 查看最新结果
JSON.parse(localStorage.getItem('latestResult'))

// 统计结局分布
const results = JSON.parse(localStorage.getItem('gameResults') || '[]')
const distribution = {}
results.forEach(r => {
  distribution[r.endingId] = (distribution[r.endingId] || 0) + 1
})
console.table(distribution)
```
